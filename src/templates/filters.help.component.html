Filters help

Filter name Description Example(s)
any Tries to match as a substring in any of the tags. This is also the default filter, the compiler falls back to using it when the expression provided does not match anything. any:foobar
foobar
meta Matches against the specified member of the CatalogFile interface. meta:tags.content.type=image/*
id Matches against the unique ID of the catalog. id:http://127.0.0.1:9000/bucket/subdir
filesize Matches against the filesize. Numeric operators can be used, typical units can be used at the number specification. filesize <= 5M
width Matches against the width of the media content. Numeric operators can be used. width >= 1024
height Matches against the height of the media content. Numeric operators can be used. height >= 768
duration Matches against the height of the duration (second) of the video content. Numeric operators can be used. Filters out non-video content. duration >= 60
tag Fine matching against tags as string. Wildcards are supported (by default partial matches are discarded). Find files having exact filename some.jpg: tag:content.filename=some.jpg
Find any files starting with name some: tag:content.filename=some*
Find any files having the word some in their name: tag:content.filename=*some*
Find any content having any tags starting with family: tag:family*=*
not Negates the next part of the filter expression. not tag:family*=*
date
date.any		 Matches against any of the date tags (upload, creation, mtime). Date target needs to be specified as yyyy-mm-dd. date >= 2019-11-01
date.upload
date.creation
date.mtime		 Matches against the upload, cration or last modification date of the content, respectively. See also: date, date.any
yearmonth
yearmonth.any		 Matches against any of the year and month portion of date tags (upload, creation, mtime). Date target needs to be specified as yyyy-mm. yearmonth >= 2019-11
yearmonth.upload
yearmonth.creation
yearmonth.mtime		 Matches against the year and month portion of the upload, cration or last modification date of the content, respectively. See also: yearmonth, yearmonth.any
year
year.any		 Matches against any of the year portion of date tags (upload, creation, mtime). Date target needs to be specified as yyyy. year >= 2019
year.upload
year.creation
year.mtime		 Matches against the year of the upload, cration or last modification date of the content, respectively. See also: year, year.any
month
month.any		 Matches against any of the month portion of date tags (upload, creation, mtime). Date target needs to be specified as mm. month >= 11
month.upload
month.creation
month.mtime		 Matches against the month of the upload, cration or last modification date of the content, respectively. See also: month, month.any
day
day.any		 Matches against any of the day portion of date tags (upload, creation, mtime). Date target needs to be specified as dd. day >= 11
day.upload
day.creation
day.mtime		 Matches against the day of the upload, cration or last modification date of the content, respectively. See also: day, day.any
and Operator to evaluate the expression using the boolean AND conditional operator between filters. tag:content.filename=some.jpg and filesize <= 5M
or Operator to evaluate the expression using the boolean OR conditional operator between filters. tag:content.filename=some.jpg or filesize <= 5M
parentheses Operator to group together multiple filter expressions and potentially combine them together with boolean operators. Inside a parentheses block only the same boolean operator may be used. Parentheses blocks can be nested. (tag:content.filename=some.jpg and width < 2000) or (filesize <= 5M)
